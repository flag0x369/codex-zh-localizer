# Security Policy

## Reporting a Vulnerability

Please do not post vulnerabilities, tokens, private paths, screenshots with account data, or sensitive logs in public issues.

Report security concerns privately by opening a GitHub security advisory for this repository. If advisories are unavailable, open a minimal public issue that says you have a security report to share, without including sensitive details.

## Supported Versions

Security fixes are handled on the default branch. Please update to the latest published version or latest commit before reporting a suspected vulnerability.

## Scope

This project is a local CLI. It should not read or print tokens, cookies, private keys, passwords, or unrelated user data. Reports about accidental credential exposure, unsafe writes, package contents, or GitHub Actions behavior are in scope.
